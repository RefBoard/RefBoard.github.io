# 도메인 구매 및 GitHub Pages 연결 완전 가이드

## 📋 전체 과정 개요

1. **도메인 구매** (약 10-15분)
2. **GitHub Pages에 맞춤 도메인 설정** (약 5분)
3. **DNS 설정** (약 10분)
4. **SSL 인증서 자동 설정 대기** (약 10-30분)
5. **Google AdSense에 새 도메인 등록** (약 5분)

**총 소요 시간**: 약 40분 ~ 1시간
**총 비용**: 연간 약 15,000원 ~ 20,000원

## 1단계: 도메인 구매

### 1-1. 도메인 업체 선택

**한국 업체 (추천)**:
- **가비아** (Gabia): https://www.gabia.com
- **후이즈** (Whois): https://whois.co.kr
- **카페24**: https://www.cafe24.com

**해외 업체 (더 저렴할 수 있음)**:
- **Namecheap**: https://www.namecheap.com
- **Cloudflare**: https://www.cloudflare.com/products/registrar/

### 1-2. 도메인 검색 및 구매 (가비아 예시)

1. **가비아 접속**
   - https://www.gabia.com 접속
   - 상단 메뉴에서 **"도메인"** 클릭

2. **도메인 검색**
   - 검색창에 원하는 도메인 입력 (예: `refboard`)
   - **검색** 버튼 클릭

3. **도메인 선택**
   - 사용 가능한 도메인 목록 확인
   - 추천: `refboard.com` 또는 `refboard.co.kr`
   - 원하는 도메인 옆 **"장바구니"** 클릭

4. **옵션 선택**
   - **등록 기간**: 1년 선택 (기본값)
   - **WHOIS 정보 보호**: 선택사항 (연간 약 3,000원 추가)
   - **DNS 관리**: 기본값 유지 (나중에 설정 가능)

5. **결제**
   - 장바구니에서 **"결제하기"** 클릭
   - 결제 정보 입력 및 결제 완료

### 1-3. 구매 확인

- 구매 완료 후 이메일 확인
- 도메인 관리 페이지 접속 가능 여부 확인

## 2단계: GitHub Pages에 맞춤 도메인 설정

### 2-1. GitHub 리포지토리 접속

1. **리포지토리로 이동**
   - https://github.com/RefBoard/RefBoard.github.io 접속
   - (또는 본인의 리포지토리 주소)

2. **Settings 메뉴 클릭**
   - 리포지토리 상단의 **"Settings"** 탭 클릭

### 2-2. Pages 설정

1. **왼쪽 메뉴에서 "Pages" 클릭**
   - Settings 페이지 왼쪽 사이드바에서 **"Pages"** 클릭

2. **Custom domain 입력**
   - **"Custom domain"** 섹션 찾기
   - 입력창에 구매한 도메인 입력:
     - 예: `www.refboard.com` (www 포함)
     - 또는: `refboard.com` (www 없이)
   - **"Save"** 버튼 클릭

3. **경고 메시지 확인**
   - "DNS configuration needed" 경고가 표시됨 (정상)
   - 다음 단계에서 DNS 설정할 예정

### 2-3. CNAME 파일 확인 (자동 생성됨)

- GitHub가 자동으로 `CNAME` 파일을 생성합니다
- 이 파일은 리포지토리 루트에 있어야 합니다
- 파일 내용: `www.refboard.com` (또는 입력한 도메인)

## 3단계: DNS 설정

### 3-1. 도메인 관리 페이지 접속

1. **도메인 업체 사이트 로그인**
   - 가비아 (또는 구매한 업체) 접속
   - 로그인 후 **"도메인 관리"** 또는 **"My Domains"** 메뉴 클릭

2. **DNS 설정 페이지로 이동**
   - 구매한 도메인 선택
   - **"DNS 관리"** 또는 **"DNS 설정"** 클릭

### 3-2. DNS 레코드 추가

**옵션 A: www 서브도메인 사용 (권장)**

1. **CNAME 레코드 추가**
   - 레코드 타입: **CNAME**
   - 호스트/이름: `www`
   - 값/대상: `refboard.github.io` (GitHub 사용자명으로 변경)
   - TTL: `3600` (또는 기본값)
   - **저장** 또는 **추가** 클릭

2. **A 레코드 추가 (루트 도메인용)**
   - 레코드 타입: **A**
   - 호스트/이름: `@` 또는 공백 (루트 도메인)
   - 값/대상: GitHub Pages IP 주소 (아래 중 하나):
     ```
     185.199.108.153
     185.199.109.153
     185.199.110.153
     185.199.111.153
     ```
   - 4개의 A 레코드를 모두 추가 (각각 다른 IP)
   - TTL: `3600`
   - **저장** 클릭

**옵션 B: www 없이 루트 도메인만 사용**

1. **A 레코드만 추가**
   - 위의 4개 A 레코드만 추가
   - CNAME 레코드는 추가하지 않음

### 3-3. DNS 설정 확인

- 설정 저장 후 **"적용"** 또는 **"저장"** 버튼 클릭
- 변경사항이 반영되는데 **10분 ~ 24시간** 소요될 수 있습니다

## 4단계: DNS 전파 확인

### 4-1. 온라인 도구로 확인

1. **DNS 확인 도구 사용**
   - https://dnschecker.org 접속
   - 도메인 입력 (예: `refboard.com`)
   - 레코드 타입: `A` 또는 `CNAME` 선택
   - **"Search"** 클릭
   - 전 세계 DNS 서버에서 확인 결과 표시

2. **확인 사항**
   - CNAME 레코드: `refboard.github.io`로 설정되어 있는지 확인
   - A 레코드: GitHub IP 주소로 설정되어 있는지 확인

### 4-2. 명령 프롬프트로 확인 (Windows)

```cmd
# CNAME 레코드 확인
nslookup www.refboard.com

# A 레코드 확인
nslookup refboard.com
```

### 4-3. GitHub Pages 설정 확인

1. **GitHub 리포지토리 Settings > Pages로 이동**
2. **Custom domain 섹션 확인**
   - 도메인 옆에 **"DNS check successful"** 또는 **"✓"** 표시 확인
   - **"Enforce HTTPS"** 체크박스가 활성화되면 체크

## 5단계: SSL 인증서 자동 설정

### 5-1. 자동 설정 대기

- GitHub Pages가 자동으로 SSL 인증서를 발급합니다
- **10분 ~ 30분** 정도 소요됩니다
- "Enforce HTTPS" 옵션이 활성화되면 완료된 것입니다

### 5-2. HTTPS 접속 확인

1. **브라우저에서 접속**
   - `https://www.refboard.com` 접속
   - 또는 `https://refboard.com` 접속

2. **자물쇠 아이콘 확인**
   - 주소창 왼쪽에 **자물쇠 아이콘** 표시 확인
   - "연결이 안전합니다" 메시지 확인

## 6단계: Google AdSense에 새 도메인 등록

### 6-1. Google AdSense 접속

1. **AdSense 대시보드 접속**
   - https://www.google.com/adsense 접속
   - 로그인

2. **사이트 관리**
   - 왼쪽 메뉴에서 **"사이트"** 클릭
   - 또는 **"사이트 추가"** 클릭

### 6-2. 새 사이트 추가

1. **사이트 URL 입력**
   - 새 도메인 입력: `https://www.refboard.com`
   - 또는: `https://refboard.com`

2. **사이트 추가 확인**
   - **"사이트 추가"** 또는 **"추가"** 클릭

### 6-3. ads.txt 파일 확인

1. **ads.txt 파일 위치 확인**
   - `https://www.refboard.com/ads.txt` 접속
   - 또는 `https://refboard.com/ads.txt` 접속
   - 내용이 올바르게 표시되는지 확인

2. **AdSense에서 확인 요청**
   - AdSense 대시보드에서 **"ads.txt 다시 확인"** 클릭
   - 또는 자동으로 확인될 때까지 대기 (최대 24시간)

## 7단계: 홈페이지 URL 업데이트

### 7-1. index.html 수정

도메인이 변경되었으므로 홈페이지의 Open Graph 태그와 링크를 업데이트해야 합니다.

**수정할 파일**: `D:\Scripts\RefBoard\refboard.github.io\index.html`

**수정할 부분**:
```html
<!-- 기존 -->
<meta property="og:url" content="https://trackk81.github.io/" />
<meta property="og:image" content="https://trackk81.github.io/images/og-image.png" />

<!-- 변경 후 -->
<meta property="og:url" content="https://www.refboard.com/" />
<meta property="og:image" content="https://www.refboard.com/images/og-image.png" />
```

### 7-2. robots.txt 수정

**수정할 파일**: `D:\Scripts\RefBoard\refboard.github.io\robots.txt`

```txt
# 기존
# Sitemap: https://refboard.github.io/sitemap.xml

# 변경 후
Sitemap: https://www.refboard.com/sitemap.xml
```

### 7-3. 변경사항 커밋 및 푸시

```bash
cd D:\Scripts\RefBoard\refboard.github.io
git add index.html robots.txt
git commit -m "Update URLs to custom domain"
git push origin main
```

## 🆘 문제 해결

### DNS가 전파되지 않는 경우

1. **대기 시간 확인**
   - DNS 전파는 최대 24시간 소요될 수 있습니다
   - 보통 10분 ~ 1시간 내에 완료됩니다

2. **DNS 설정 재확인**
   - 도메인 관리 페이지에서 설정이 올바른지 확인
   - 오타가 없는지 확인

3. **브라우저 캐시 삭제**
   - 브라우저 캐시를 삭제하고 다시 시도
   - 시크릿 모드에서 확인

### SSL 인증서가 발급되지 않는 경우

1. **DNS 전파 완료 대기**
   - DNS가 완전히 전파된 후 SSL이 발급됩니다
   - 최대 24시간 소요될 수 있습니다

2. **GitHub Pages 설정 확인**
   - Custom domain이 올바르게 입력되어 있는지 확인
   - CNAME 파일이 리포지토리 루트에 있는지 확인

### Google AdSense에서 ads.txt를 찾을 수 없는 경우

1. **ads.txt 파일 확인**
   - `https://www.refboard.com/ads.txt` 직접 접속
   - 내용이 올바르게 표시되는지 확인

2. **AdSense에서 수동 확인 요청**
   - AdSense 대시보드에서 "ads.txt 다시 확인" 클릭
   - 최대 24시간 대기
