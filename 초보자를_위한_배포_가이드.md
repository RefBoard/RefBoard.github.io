# 초보자를 위한 RefBoard 배포 가이드

## 🎯 목표
RefBoard 앱을 GitHub Pages에 배포해서 웹에서 사용할 수 있게 만들기

---

## 📚 기본 개념 이해하기

### Git이란?
- Git은 파일의 변경사항을 추적하는 프로그램입니다
- GitHub는 Git을 사용하는 온라인 저장소입니다
- 리포지토리(Repository) = 프로젝트 폴더 + Git 관리 기능

### 현재 상황
- `D:\Scripts\refboard-website` 폴더가 있지만
- 이 폴더가 Git 리포지토리가 아니어서 `git status` 명령이 작동하지 않습니다

---

## 🔍 문제 확인하기

### 1단계: 폴더가 있는지 확인

**Windows 탐색기**를 열어서 확인하세요:
1. `D:\Scripts` 폴더를 엽니다
2. `refboard-website` 폴더가 있는지 확인합니다

**두 가지 경우가 있습니다:**

#### 경우 1: 폴더가 아예 없는 경우
→ 리포지토리를 클론해야 합니다 (아래 "해결 방법 1" 참고)

#### 경우 2: 폴더는 있지만 Git 리포지토리가 아닌 경우
→ 폴더 안에 `.git` 폴더가 없습니다
→ 리포지토리를 다시 클론하거나 초기화해야 합니다

---

## ✅ 해결 방법

### 방법 1: GitHub에서 리포지토리 클론하기 (권장)

**단계별 설명:**

#### 1-1. CMD 창 열기
- Windows 키를 누릅니다
- `cmd`를 입력하고 Enter를 누릅니다
- 검은색 창(CMD)이 열립니다

#### 1-2. D:\Scripts 폴더로 이동
CMD 창에 다음을 입력하고 Enter:
```
cd D:\Scripts
```

#### 1-3. 기존 폴더가 있다면 삭제 (선택사항)
만약 `refboard-website` 폴더가 이미 있다면:
```
rmdir /s /q refboard-website
```
(이 명령은 폴더를 완전히 삭제합니다. 주의하세요!)

#### 1-4. GitHub에서 리포지토리 클론
CMD 창에 다음을 입력하고 Enter:
```
git clone https://github.com/trackk81/refboard-website.git
```

**무슨 일이 일어나는가?**
- GitHub에서 `refboard-website` 리포지토리를 다운로드합니다
- `D:\Scripts\refboard-website` 폴더가 생성됩니다
- 이 폴더 안에 `.git` 폴더가 자동으로 생성됩니다 (숨김 폴더)

#### 1-5. 완료 확인
```
cd refboard-website
git status
```

**성공하면:**
```
On branch main
nothing to commit, working tree clean
```
이런 메시지가 나옵니다.

**실패하면:**
- 인터넷 연결 확인
- GitHub 계정 확인
- 리포지토리 이름 확인 (`trackk81/refboard-website`)

---

### 방법 2: 폴더를 Git 리포지토리로 초기화하기

만약 폴더가 이미 있고 파일들이 있다면:

#### 2-1. CMD에서 폴더로 이동
```
cd D:\Scripts\refboard-website
```

#### 2-2. Git 초기화
```
git init
```

#### 2-3. 원격 리포지토리 연결
```
git remote add origin https://github.com/trackk81/refboard-website.git
```

#### 2-4. 원격 리포지토리에서 가져오기
```
git pull origin main
```

## 📦 파일 복사하기

리포지토리가 준비되면 빌드한 파일을 복사해야 합니다.

### 방법 A: Windows 탐색기 사용 (가장 쉬움) ⭐

1. **빌드된 파일이 있는 폴더 열기**
   - `D:\Scripts\RefBoard\dist-web` 폴더를 탐색기에서 엽니다
   - 이 폴더 안에 `index.html`, `assets` 폴더 등이 보입니다

2. **모든 파일 선택**
   - `Ctrl + A` 키를 누릅니다 (모두 선택)
   - 모든 파일이 파란색으로 선택됩니다

3. **복사**
   - `Ctrl + C` 키를 누릅니다 (복사)

4. **리포지토리 폴더로 이동**
   - `D:\Scripts\refboard-website` 폴더를 탐색기에서 엽니다

5. **refboard 폴더 만들기**
   - 폴더 안에서 마우스 우클릭
   - "새로 만들기" → "폴더"
   - 폴더 이름을 `refboard`로 입력

6. **refboard 폴더 열기**
   - `refboard` 폴더를 더블클릭해서 엽니다

7. **붙여넣기**
   - `Ctrl + V` 키를 누릅니다 (붙여넣기)
   - 파일들이 복사됩니다

### 방법 B: CMD 사용

```
cd D:\Scripts\refboard-website
mkdir refboard
xcopy /E /I /Y D:\Scripts\RefBoard\dist-web\* refboard\
```

## 🚀 GitHub에 업로드하기

### 1단계: 리포지토리 폴더로 이동
CMD 창에서:
```
cd D:\Scripts\refboard-website
```

### 2단계: 현재 상태 확인
```
git status
```

**보이는 내용:**
```
On branch main
Untracked files:
  refboard/
```
이런 식으로 `refboard` 폴더가 보여야 합니다.

### 3단계: 파일 추가
```
git add refboard/
```

**무슨 일이 일어나는가?**
- Git이 `refboard` 폴더의 변경사항을 추적하기 시작합니다
- 아직 업로드되지는 않았습니다

### 4단계: 변경사항 저장 (커밋)
```
git commit -m "Deploy RefBoard app to /refboard folder"
```

**무슨 일이 일어나는가?**
- 변경사항을 로컬에 저장합니다
- "Deploy RefBoard app to /refboard folder"는 저장 메시지입니다

### 5단계: GitHub에 업로드 (푸시)
```
git push origin main
```

**무슨 일이 일어나는가?**
- 로컬의 변경사항을 GitHub에 업로드합니다
- GitHub 사용자 이름과 비밀번호(또는 토큰)를 입력해야 할 수 있습니다

**인증이 필요한 경우:**
- GitHub 사용자 이름 입력
- 비밀번호 대신 Personal Access Token을 사용해야 할 수 있습니다
- 토큰이 없다면: GitHub → Settings → Developer settings → Personal access tokens → Generate new token

## ✅ 완료 확인

### 1. GitHub 웹사이트에서 확인
1. 브라우저에서 `https://github.com/trackk81/refboard-website` 접속
2. `refboard` 폴더가 보이는지 확인
3. `refboard/index.html` 파일이 있는지 확인

### 2. GitHub Pages 설정
1. 리포지토리 페이지에서 "Settings" 클릭
2. 왼쪽 메뉴에서 "Pages" 클릭
3. Source를 "Deploy from a branch" 선택
4. Branch를 "main" 선택
5. Folder를 "/ (root)" 선택
6. "Save" 클릭

### 3. 배포된 사이트 확인
몇 분 후 (보통 1-5분):
- `https://trackk81.github.io/refboard-website/refboard/` 접속
- RefBoard 앱이 로드되는지 확인

## 🆘 문제 해결

### "fatal: not a git repository" 에러
→ 리포지토리를 클론하지 않았거나, 잘못된 폴더에 있습니다
→ 해결: "방법 1: GitHub에서 리포지토리 클론하기" 참고

### "git clone"이 안 되는 경우
→ 인터넷 연결 확인
→ GitHub 계정 확인
→ 리포지토리가 존재하는지 확인

### "git push"가 안 되는 경우
→ GitHub 인증 확인
→ Personal Access Token 사용
→ 리포지토리 권한 확인

### 파일이 복사되지 않는 경우
→ Windows 탐색기를 사용하세요 (가장 확실함)
→ `dist-web` 폴더에 파일이 있는지 확인
