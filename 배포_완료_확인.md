## ⏰ 배포 확인 (중요!)

GitHub Pages 배포에는 **1-5분** 정도 걸릴 수 있습니다.

### 1단계: GitHub Pages 배포 상태 확인

1. **리포지토리 접속**
   - `https://github.com/trackk81/refboard-website` 접속

2. **Settings > Pages 확인**
   - 리포지토리 상단의 "Settings" 클릭
   - 왼쪽 메뉴에서 "Pages" 클릭
   - "Your site is live at..." 메시지 확인
   - 최근 배포 시간 확인

3. **Actions 탭 확인 (선택사항)**
   - 리포지토리 상단의 "Actions" 클릭
   - 최근 워크플로우 실행 상태 확인

## 🔍 사이트 접속 확인

### 방법 1: 일반 브라우저

1. **브라우저에서 접속**
   - `https://trackk81.github.io/refboard-website/` 접속

2. **개발자 도구 열기**
   - `F12` 키 누르기

3. **Console 탭 확인**
   - 에러가 없는지 확인
   - RefBoard 앱이 로드되는지 확인

4. **Network 탭 확인**
   - 파일들이 제대로 로드되는지 확인
   - 404 에러가 없는지 확인

### 방법 2: 시크릿 모드 (캐시 없이 테스트)

1. **시크릿 모드 열기**
   - Chrome: `Ctrl + Shift + N`
   - Edge: `Ctrl + Shift + N`
   - Firefox: `Ctrl + Shift + P`

2. **URL 접속**
   - `https://trackk81.github.io/refboard-website/` 접속

3. **확인**
   - RefBoard 앱이 정상적으로 로드되는지 확인

## 🐛 문제 해결

### 여전히 404 에러가 발생하는 경우

1. **브라우저 캐시 삭제**
   - `Ctrl + Shift + Delete`
   - "캐시된 이미지 및 파일" 선택
   - 삭제

2. **강력 새로고침**
   - `Ctrl + Shift + R` (Windows)
   - `Cmd + Shift + R` (Mac)

3. **몇 분 더 기다리기**
   - GitHub Pages 배포에 시간이 걸릴 수 있습니다
   - 최대 5-10분 기다려보세요

### 빈 페이지가 표시되는 경우

1. **브라우저 콘솔 확인**
   - `F12` > Console 탭
   - 에러 메시지 확인

2. **Network 탭 확인**
   - `F12` > Network 탭
   - 실패한 파일 확인
   - 파일 경로 확인

3. **GitHub 리포지토리 확인**
   - `https://github.com/trackk81/refboard-website` 접속
   - 루트에 `index.html`이 있는지 확인
   - `assets` 폴더가 있는지 확인

### Firebase/Google 인증 에러

1. **콘솔 에러 확인**
   - Firebase 설정이 올바른지 확인
   - Google Drive API 설정 확인

2. **환경 변수 확인**
   - `.env` 파일 확인
   - Firebase 프로젝트 설정 확인
