# 404 에러 해결 가이드

## 🔍 문제 원인

현재 `index.html`에서 파일 경로가 `/refboard/assets/...`로 설정되어 있는데, 실제 GitHub Pages URL은 `https://trackk81.github.io/refboard-website/refboard/`입니다.

**문제:**
- `index.html` 경로: `/refboard/assets/index-B0p4sZXv.js`
- 실제 접근 URL: `https://trackk81.github.io/refboard/assets/...` (잘못됨)
- 올바른 URL: `https://trackk81.github.io/refboard-website/refboard/assets/...`

---

## ✅ 해결 방법: 루트에 배포

가장 간단한 해결책은 **루트에 배포**하는 것입니다.

### 1단계: 설정 변경 (이미 완료됨)

`vite.config.web.ts`의 `base`를 `/`로 변경했습니다.

### 2단계: 다시 빌드

CMD에서:
```bash
cd D:\Scripts\RefBoard
npm run build:web
```

### 3단계: 파일 복사

Windows 탐색기에서:
1. `D:\Scripts\RefBoard\dist-web` 폴더 열기
2. 모든 파일 선택 (`Ctrl + A`)
3. 복사 (`Ctrl + C`)
4. `D:\Scripts\refboard-website` 폴더로 이동 (루트로!)
5. 붙여넣기 (`Ctrl + V`)

**중요:** `refboard` 폴더가 아니라 **루트**에 복사하세요!

### 4단계: 기존 refboard 폴더 삭제 (선택사항)

기존 `refboard` 폴더를 삭제하려면:
```bash
cd D:\Scripts\refboard-website
rmdir /s /q refboard
```

또는 Windows 탐색기에서 `refboard` 폴더를 삭제하세요.

### 5단계: Git 작업

CMD에서:
```bash
cd D:\Scripts\refboard-website

# 모든 변경사항 추가
git add .

# 커밋
git commit -m "Deploy RefBoard to root (fix 404 errors)"

# 푸시
git push origin main
```

## 🔄 대안: refboard 폴더 유지하면서 수정

만약 `refboard` 폴더에 계속 배포하고 싶다면:

### 방법 1: base를 전체 경로로 변경

`vite.config.web.ts`:
```typescript
base: '/refboard-website/refboard/',
```

하지만 이 방법은 리포지토리 이름이 바뀌면 문제가 됩니다.

### 방법 2: 상대 경로 사용

`vite.config.web.ts`:
```typescript
base: './',
```

하지만 이 방법도 일부 경우에 문제가 될 수 있습니다.

**추천:** 루트에 배포하는 것이 가장 안정적입니다!

## 🆘 여전히 문제가 있다면

1. **브라우저 캐시 삭제**
   - `Ctrl + Shift + Delete`
   - 캐시된 이미지 및 파일 삭제

2. **시크릿 모드에서 테스트**
   - `Ctrl + Shift + N` (Chrome)
   - 캐시 없이 테스트

3. **GitHub Pages 배포 확인**
   - 리포지토리 Settings > Pages
   - 배포 상태 확인
   - 몇 분 기다리기 (배포에 시간이 걸릴 수 있음)
