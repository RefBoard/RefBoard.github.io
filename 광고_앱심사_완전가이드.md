# 데스크톱 앱 광고 통합 및 앱 심사 완전 가이드

## 🎯 핵심 질문: 순서는?

### ✅ 권장 순서: 광고 먼저 → 앱 심사 나중

**이유:**
1. 광고 승인에는 1-2주가 걸림
2. 앱 심사는 광고가 없어도 가능하지만, 광고를 먼저 통합하면 한 번에 제출 가능
3. 앱 심사 중 광고를 추가하면 재심사가 필요할 수 있음

**하지만:** 앱 심사를 먼저 통과한 후 광고를 추가하는 것도 가능합니다!

---

## 📋 전체 순서 (단계별)

### 1단계: 광고 플랫폼 신청 (1-2주) ⏰

#### Carbon Ads 신청 (권장)

1. **웹사이트 접속**
   ```
   https://www.carbonads.net/
   ```

2. **신청서 작성**
   - "Publishers" → "Apply to be a Publisher" 클릭
   - **Website URL**: `https://trackk81.github.io/refboard-website/`
   - **Monthly Visitors**: 예상치 입력 (예: 10,000-50,000)
   - **Category**: Development Tools, Design Tools
   - **Description** (영어):
     ```
     RefBoard is a visual reference board desktop application for designers and developers.
     Features: Real-time collaboration, Google Drive integration, PSD file support, 
     infinite canvas, drawing tools. Target audience: UI/UX designers, developers, 
     digital artists, creative professionals.
     ```

3. **승인 대기**
   - 보통 1-2주 소요
   - 이메일로 승인/거부 알림

#### 승인 후 받을 것
- **Zone ID**: 예) `CE7DT2QY`
- 광고 코드 예시

---

### 2단계: 광고 코드 통합 (1일) 🔧

#### Carbon Ads 승인 후

1. **Zone ID 확인**
   - Carbon Ads 대시보드에서 Zone ID 복사

2. **`public/carbon-ad.html` 파일 수정**

   **현재 상태 (102-107줄):**
   ```html
   <!-- 
   <script async type="text/javascript" 
           src="//cdn.carbonads.com/carbon.js?serve=YOUR_ZONE_ID&placement=refboard" 
           id="_carbonads_js">
   </script>
   -->
   ```

   **수정 후:**
   ```html
   <script async type="text/javascript" 
           src="//cdn.carbonads.com/carbon.js?serve=CE7DT2QY&placement=refboard" 
           id="_carbonads_js">
   </script>
   ```
   ⚠️ `CE7DT2QY`를 실제 Zone ID로 교체하세요!

3. **플레이스홀더 제거 (선택사항)**
   - 110-118줄의 플레이스홀더 div는 그대로 두어도 됨
   - 광고가 로드되면 자동으로 숨겨짐

4. **테스트**
   ```bash
   npm run electron:dev
   ```
   - 앱 실행 후 2시간 작업 시뮬레이션
   - 광고가 표시되는지 확인

---

### 3단계: 앱 빌드 및 최종 테스트 (1일) 📦

```bash
# 프로덕션 빌드
npm run electron:build
```

**확인 사항:**
- [ ] 설치 파일 생성됨 (`release/RefBoard Setup 0.0.0.exe`)
- [ ] 설치 후 앱 실행 정상
- [ ] 광고가 2시간 후 표시됨
- [ ] 광고 클릭이 작동함

---

### 4단계: Microsoft Store 앱 심사 준비 (2-3일) 🏪

#### A. Microsoft Partner Center 계정 생성

1. **웹사이트 접속**
   ```
   https://partner.microsoft.com/dashboard
   ```

2. **개발자 계정 등록**
   - 비용: **$19 (일회성)** 또는 **$99/년**
   - 결제 정보 입력
   - 신원 확인 (신분증 사진 업로드)
   - 승인 대기: 1-3일

#### B. 앱 정보 준비

**필수 파일:**
1. **앱 아이콘**
   - ✅ 이미 준비됨: `icon/RB_icon.png`
   - 필요 시 256x256, 512x512 버전 생성

2. **스크린샷** (최소 1개, 권장 4-9개)
   - 해상도: 최소 1366x768, 권장 1920x1080
   - 형식: PNG 또는 JPEG
   - 내용:
     - 메인 화면 (보드 화면)
     - 이미지 추가 화면
     - 협업 기능 화면
     - 설정 화면

3. **앱 설명** (한국어 및 영어)

   **한국어:**
   ```
   RefBoard는 디자이너와 개발자를 위한 시각적 참조 보드 데스크톱 애플리케이션입니다.

   주요 기능:
   • 무한 캔버스에서 이미지, 동영상, PSD 파일 배치
   • 실시간 협업 기능
   • Google Drive 통합
   • 펜 도구 및 텍스트 편집
   • 그룹화 및 화살표 연결

   타겟 사용자:
   UI/UX 디자이너, 개발자, 디지털 아티스트, 크리에이티브 전문가
   ```

   **영어:**
   ```
   RefBoard is a visual reference board desktop application for designers and developers.

   Key Features:
   • Infinite canvas for images, videos, and PSD files
   • Real-time collaboration
   • Google Drive integration
   • Pen tools and text editing
   • Grouping and arrow connections

   Target Users:
   UI/UX designers, developers, digital artists, creative professionals
   ```

4. **개인정보 처리방침**
   - URL: `https://trackk81.github.io/refboard-website/privacy.html`
   - 또는 텍스트로 작성

#### C. 앱 패키지 준비

**현재 설정 (NSIS 설치 파일):**
- ✅ 이미 설정됨: `package.json`의 `build` 섹션
- Windows 설치 파일 (.exe) 생성됨

**Microsoft Store용 (선택사항):**
- MSIX 패키지로 변환 가능하지만, 현재 NSIS 설치 파일로도 배포 가능
- Microsoft Store 외부 배포 가능

---

### 5단계: Microsoft Partner Center에서 제출 (1일) 📤

#### 1. 새 앱 만들기

1. Partner Center 대시보드 접속
2. "앱 만들기" 클릭
3. 앱 이름: `RefBoard`

#### 2. 앱 정보 입력

**기본 정보:**
- 앱 이름: RefBoard
- 앱 설명: 위에서 준비한 설명 입력
- 카테고리: 생산성 도구 (Productivity)
- 가격: 무료 (Free)
- 연령 등급: E (Everyone)

**스토어 등록 정보:**
- 앱 아이콘 업로드 (256x256, 512x512)
- 스크린샷 업로드 (최소 1개)
- 기능 그래픽 (선택사항, 1920x1080)

**가격 및 사용 가능성:**
- 가격: 무료
- 국가: 전 세계 또는 특정 국가 선택

**개인정보 보호:**
- 개인정보 처리방침 URL: `https://trackk81.github.io/refboard-website/privacy.html`
- 데이터 수집 명시:
  - Google 계정 정보 (로그인용)
  - Google Drive 접근 (파일 저장용)
  - 사용자 이메일 (협업용)

#### 3. 패키지 업로드

**방법 1: 일반 설치 파일 (현재 설정)**
- NSIS 설치 파일 업로드
- Microsoft Store 외부 배포

**방법 2: MSIX 패키지 (Microsoft Store 내부)**
- MSIX 패키지 생성 필요
- 더 복잡하지만 Store 내부 배포 가능

#### 4. 제출

1. 모든 필수 정보 입력 확인
2. "제출" 클릭
3. 심사 대기 (1-3일)

---

### 6단계: 심사 대기 및 승인 (1-3일) ⏳

- **심사 기간**: 보통 1-3일
- **상태 확인**: Partner Center 대시보드
- **피드백**: 문제가 있으면 이메일로 알림

**승인 후:**
- 앱이 Microsoft Store에 자동 게시됨
- 사용자가 다운로드 가능

## 💡 중요 팁

### 광고 관련

1. **광고는 나중에 추가 가능**
   - 앱 심사를 먼저 통과한 후 광고 추가 가능
   - 하지만 광고를 먼저 통합하면 한 번에 제출 가능

2. **투명성**
   - 앱 설명에 광고 표시를 명시하는 것이 좋음
   - "2시간마다 15분 동안 광고가 표시됩니다" 같은 설명

3. **사용자 경험**
   - 현재 구현이 좋음: 2시간마다 15분만 표시
   - 광고가 앱 사용을 방해하지 않음

### 앱 심사 관련

1. **개인정보 처리방침 필수**
   - Google 계정 정보 사용 명시
   - Google Drive 접근 명시
   - 데이터 수집 목적 명시

2. **기능 설명 명확히**
   - 모든 기능을 정확히 설명
   - 필요한 권한 명시

3. **스크린샷 품질**
   - 고해상도 사용
   - 실제 앱 화면 사용

## 🆘 자주 묻는 질문

### Q1: 광고를 먼저 달아야 하나요, 앱 심사를 먼저 해야 하나요?

**A:** 둘 다 가능합니다!
- **광고 먼저**: 한 번에 제출 가능, 하지만 광고 승인 대기 필요
- **앱 심사 먼저**: 빠르게 스토어에 게시 가능, 나중에 광고 추가 시 업데이트 필요

**권장:** 앱 심사를 먼저 통과한 후 광고 추가 (더 빠름)

### Q2: 앱 심사에서 광고가 거부될 수 있나요?

**A:** 일반적으로 거부되지 않습니다. 다만:
- 광고가 앱 사용을 방해하지 않아야 함
- 광고 클릭을 강제하지 않아야 함
- 개인정보 처리방침에 광고 관련 내용 명시 (선택사항)

### Q3: Microsoft Store 외부 배포도 가능한가요?

**A:** 네, 가능합니다!
- 현재 NSIS 설치 파일로 배포 가능
- GitHub Releases에 업로드 가능
- 직접 배포 가능

### Q4: 광고 수익은 언제부터 발생하나요?

**A:** 
- Carbon Ads 승인 후 즉시 시작
- 광고 클릭 시 수익 발생
- 월별 정산
