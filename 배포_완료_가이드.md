# 배포 완료 가이드

## ✅ 현재 상황
파일이 `refboard-website` 리포지토리의 **루트**에 복사되었습니다.

보이는 파일들:
- `index.html` (수정됨)
- `404.html` (새 파일)
- `assets/` 폴더 (새 폴더)
- `carbon-ad.html` (새 파일)

---

## 🎯 두 가지 선택지

### 옵션 1: 루트에 배포 (더 간단함) ⭐

현재 파일이 루트에 있으므로, 이대로 배포하는 것이 가장 간단합니다.

**필요한 작업:**
1. `vite.config.web.ts`의 `base`를 `/`로 변경
2. 다시 빌드
3. 파일 다시 복사
4. Git 커밋 및 푸시

**배포 URL:**
- `https://trackk81.github.io/refboard-website/`

**장점:**
- 기존 `index.html`을 RefBoard 앱으로 대체
- URL이 간단함
- 설정이 간단함

**단점:**
- 기존 웹사이트가 사라짐

---

### 옵션 2: refboard 폴더에 배포 (기존 사이트 유지)

기존 웹사이트를 유지하고 RefBoard를 서브폴더에 배포합니다.

**필요한 작업:**
1. 현재 루트의 파일들을 `refboard` 폴더로 이동
2. Git 커밋 및 푸시

**배포 URL:**
- `https://trackk81.github.io/refboard-website/refboard/`

**장점:**
- 기존 웹사이트 유지
- RefBoard는 별도 경로에서 접근

**단점:**
- URL이 더 김

---

## 🚀 빠른 배포 (옵션 1: 루트에 배포)

### 1단계: 설정 변경

`vite.config.web.ts` 파일을 열고:
```typescript
base: '/',  // 루트에 배포
```

### 2단계: 다시 빌드

CMD에서:
```bash
cd D:\Scripts\RefBoard
npm run build:web
```

### 3단계: 파일 다시 복사

Windows 탐색기에서:
1. `D:\Scripts\RefBoard\dist-web` 폴더 열기
2. 모든 파일 선택 (`Ctrl + A`)
3. 복사 (`Ctrl + C`)
4. `D:\Scripts\refboard-website` 폴더로 이동
5. 기존 파일 덮어쓰기 (`Ctrl + V`)

### 4단계: Git 작업

CMD에서:
```bash
cd D:\Scripts\refboard-website

# 모든 변경사항 추가
git add .

# 커밋
git commit -m "Deploy RefBoard app to root"

# 푸시
git push origin main
```

## 🚀 빠른 배포 (옵션 2: refboard 폴더에 배포)

### 1단계: 파일 이동

CMD에서:
```bash
cd D:\Scripts\refboard-website

# refboard 폴더 생성
mkdir refboard

# 파일 이동
move index.html refboard\
move 404.html refboard\
move assets refboard\
move carbon-ad.html refboard\
```

또는 Windows 탐색기에서:
1. `D:\Scripts\refboard-website` 폴더 열기
2. `refboard` 폴더 만들기
3. `index.html`, `404.html`, `assets`, `carbon-ad.html`을 `refboard` 폴더로 이동

### 2단계: Git 작업

CMD에서:
```bash
cd D:\Scripts\refboard-website

# 모든 변경사항 추가
git add .

# 커밋
git commit -m "Deploy RefBoard app to /refboard folder"

# 푸시
git push origin main
```

## 🆘 문제 해결

### "git add" 후에도 파일이 보이지 않는 경우
→ 파일이 제대로 복사되었는지 확인
→ `git status`로 다시 확인

### 푸시가 안 되는 경우
→ GitHub 인증 확인
→ Personal Access Token 사용

### 페이지가 로드되지 않는 경우
→ GitHub Pages 설정 확인 (Settings > Pages)
→ 브라우저 콘솔에서 에러 확인
