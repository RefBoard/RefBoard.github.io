## 📋 순서 요약

**중요: 광고를 먼저 통합하고 테스트한 후 앱 심사를 제출하세요!**

1. ✅ **광고 플랫폼 신청** (1-2주 소요)
2. ✅ **광고 코드 통합 및 테스트** (1일)
3. ✅ **앱 빌드 및 테스트** (1일)

## 🎯 1단계: 광고 플랫폼 선택 및 신청

### 옵션 1: Carbon Ads (권장) ⭐

#### 장점
- 개발자 친화적
- 깔끔한 광고 디자인
- 높은 클릭률
- 데스크톱 앱 지원

#### 신청 방법

1. **웹사이트 접속**
   - https://www.carbonads.net/
   - "Publishers" 섹션 클릭

2. **신청서 작성**
   - **Website URL**: `https://trackk81.github.io/refboard-website/` (또는 GitHub Pages URL)
   - **Monthly Visitors**: 
     - 실제 트래픽이 적어도 신청 가능
     - 예상치를 적당히 입력 (예: 10,000-50,000)
   - **Category**: 
     - Development Tools
     - Design Tools
   - **Description** (영어로 작성):
     ```
     RefBoard is a visual reference board desktop application for designers and developers.
     It allows users to collect, organize, and reference visual materials (images, videos, PSD files) 
     while working. Features include real-time collaboration, Google Drive integration, and drawing tools.
     
     Target audience: UI/UX designers, developers, digital artists, creative professionals.
     Monthly active users: Growing user base of creative professionals.
     ```

3. **승인 대기**
   - 보통 1-2주 소요
   - 이메일로 승인/거부 알림

#### 승인 후 통합

Carbon Ads에서 **Zone ID**를 받으면:

1. `public/carbon-ad.html` 파일 열기
2. 다음 코드를 찾아서 주석 해제:
```html
<!-- 현재 주석 처리된 부분 -->
<!-- 
<script async type="text/javascript" 
        src="//cdn.carbonads.com/carbon.js?serve=YOUR_ZONE_ID&placement=refboard" 
        id="_carbonads_js">
</script>
-->

<!-- 이렇게 변경 (YOUR_ZONE_ID를 실제 Zone ID로 교체) -->
<script async type="text/javascript" 
        src="//cdn.carbonads.com/carbon.js?serve=CE7DT2QY&placement=refboard" 
        id="_carbonads_js">
</script>
```

3. 플레이스홀더 제거 (선택사항):
   - `carbon-ad.html`에서 플레이스홀더 div 제거

## 🔧 2단계: 광고 코드 통합

### 현재 상태 확인

현재 `AdBanner` 컴포넌트는 이미 구현되어 있습니다:
- 위치: `src/components/AdBanner.tsx`
- 표시 조건: `showBreakReminder`가 `true`일 때 (2시간 작업 후 15분 동안)
- 광고 소스: `public/carbon-ad.html` 파일

### 통합 단계

#### 1. Carbon Ads 승인 후

1. **Zone ID 받기**
   - Carbon Ads 대시보드에서 Zone ID 확인

2. **`public/carbon-ad.html` 수정**
```html
<!DOCTYPE html>

            font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, Oxygen-Sans, Ubuntu, Cantarell, "Helvetica Neue", Helvetica, Arial, sans-serif;
            display: inline-block;
            overflow: hidden;

        #carbonads a {
            color: #fff;

            display: block;
            overflow: hidden;

            float: left;
            margin-right: 1em;
        }
        .carbon-img img {

            id="_carbonads_js">

        광고 로딩 중...
    </div>
</body>
</html>
```

   - 개발 모드에서 광고 표시 확인
   - 2시간 작업 후 광고가 나타나는지 확인

## 📦 3단계: 앱 빌드 및 테스트

### 1. 광고 통합 확인

```bash
# 개발 모드에서 테스트
npm run electron:dev
```

**확인 사항:**
- [ ] 광고가 2시간 작업 후 표시되는가?
- [ ] 광고가 15분 후 사라지는가?
- [ ] 광고 클릭이 정상 작동하는가?
- [ ] 광고가 로드되지 않을 때 에러가 없는가?

### 2. 프로덕션 빌드

```bash
# 빌드 실행
npm run electron:build
```

**빌드 결과:**
- `release/RefBoard Setup 0.0.0.exe` 생성 확인

### 3. 설치 및 테스트

1. 설치 파일 실행
2. 앱 실행 후 2시간 작업 시뮬레이션
3. 광고 표시 확인

## 🏪 4단계: Microsoft Store 앱 심사

### 준비 사항

#### 필수 파일
- [ ] 앱 아이콘: `icon/RB_icon.png` (이미 준비됨)
- [ ] 스크린샷: 최소 1개, 권장 4-9개
- [ ] 앱 설명: 한국어 및 영어
- [ ] 개인정보 처리방침: URL 또는 텍스트

#### 스크린샷 준비
- 해상도: 최소 1366x768, 권장 1920x1080
- 형식: PNG 또는 JPEG
- 내용:
  1. 메인 화면 (보드 화면)
  2. 이미지 추가 화면
  3. 협업 기능 화면
  4. 설정 화면
  5. 광고 표시 화면 (선택사항)

### Microsoft Store 등록 절차

#### 1. Microsoft Partner Center 계정 생성

1. **웹사이트 접속**
   - https://partner.microsoft.com/dashboard
   - Microsoft 계정으로 로그인

2. **개발자 계정 등록**
   - 비용: **$19 (일회성)** 또는 **$99/년**
   - 결제 정보 입력
   - 신원 확인 (신분증 사진 업로드)

3. **승인 대기**
   - 보통 1-3일 소요
   - 이메일로 승인 알림

#### 2. 앱 제출 준비

##### A. 앱 패키지 생성

**방법 1: MSIX 패키지 (권장)**

1. **Windows SDK 설치**
   - https://developer.microsoft.com/en-us/windows/downloads/windows-sdk/
   - Windows 10 SDK 이상 필요

2. **electron-builder 설정 수정**

`package.json`에 추가:
```json
{
  "build": {
    "win": {
      "target": [
        {
          "target": "msi",
          "arch": ["x64"]
        }
      ],
      "publisherName": "Your Name",
      "certificateFile": "path/to/certificate.pfx",
      "certificatePassword": "password"
    }
  }
}
```

**방법 2: 일반 설치 파일 (간단)**

현재 설정 그대로 사용:
- `nsis` 설치 프로그램 사용
- Microsoft Store 외부 배포

##### B. 앱 정보 작성

**앱 이름:**
- 한국어: RefBoard
- 영어: RefBoard

**짧은 설명:**
- 한국어: 디자이너와 개발자를 위한 시각적 참조 보드
- 영어: Visual reference board for designers and developers

**전체 설명:**
```
RefBoard는 디자이너와 개발자를 위한 시각적 참조 보드 데스크톱 애플리케이션입니다.

주요 기능:
• 무한 캔버스에서 이미지, 동영상, PSD 파일 배치
• 실시간 협업 기능
• Google Drive 통합
• 펜 도구 및 텍스트 편집
• 그룹화 및 화살표 연결

타겟 사용자:
UI/UX 디자이너, 개발자, 디지털 아티스트, 크리에이티브 전문가

RefBoard helps designers and developers collect, organize, and reference visual materials while working.

Key Features:
• Infinite canvas for images, videos, and PSD files
• Real-time collaboration
• Google Drive integration
• Pen tools and text editing
• Grouping and arrow connections

Target Users:
UI/UX designers, developers, digital artists, creative professionals
```

**카테고리:**
- 생산성 도구 (Productivity)
- 또는 디자인 도구 (Design Tools)

**가격:**
- 무료 (Free)

**연령 등급:**
- E (Everyone) - 모든 연령

#### 3. Microsoft Partner Center에서 제출

1. **새 앱 만들기**
   - Partner Center 대시보드 접속
   - "앱 만들기" 클릭
   - 앱 이름 입력: RefBoard

2. **앱 정보 입력**
   - 앱 설명
   - 스크린샷 업로드
   - 아이콘 업로드 (256x256, 512x512)
   - 기능 그래픽 (선택사항, 1920x1080)

3. **가격 및 사용 가능성**
   - 가격: 무료
   - 국가 선택: 전 세계 또는 특정 국가

4. **개인정보 보호**
   - 개인정보 처리방침 URL: 
     - `https://trackk81.github.io/refboard-website/privacy.html`
   - 데이터 수집 명시:
     - Google 계정 정보 (로그인용)
     - Google Drive 접근 (파일 저장용)
     - 사용자 이메일 (협업용)

5. **패키지 업로드**
   - MSIX 패키지 또는 설치 파일 업로드
   - 버전 정보 입력

6. **제출**
   - 모든 필수 정보 입력 확인
   - "제출" 클릭

#### 4. 심사 대기

- **심사 기간**: 보통 1-3일
- **상태 확인**: Partner Center 대시보드에서 확인
- **피드백**: 문제가 있으면 이메일로 알림

#### 5. 승인 후

- 앱이 Microsoft Store에 자동 게시됨
- 사용자가 다운로드 가능

## 🆘 문제 해결

### 광고가 표시되지 않는 경우

1. **Carbon Ads 승인 확인**
   - 이메일 확인
   - 대시보드에서 상태 확인

2. **Zone ID 확인**
   - 올바른 Zone ID인지 확인
   - `carbon-ad.html` 파일 확인

3. **네트워크 확인**
   - 인터넷 연결 확인
   - 방화벽 설정 확인

### 앱 심사 거부 시

1. **피드백 확인**
   - Partner Center에서 거부 사유 확인
   - 이메일 확인

2. **문제 수정**
   - 개인정보 처리방침 보완
   - 앱 설명 보완
   - 스크린샷 추가

3. **재제출**
   - 수정 후 다시 제출
