# 빌드 에러 해결 가이드

## 🔍 문제 원인

빌드 중 다음 에러가 발생했습니다:
```
ERROR: Cannot create symbolic link : 클라이언트에 필요한 권한이 없습니다
```

**원인:**
- Windows에서 코드 서명 도구(winCodeSign)를 압축 해제할 때 심볼릭 링크 생성이 필요함
- 심볼릭 링크 생성에는 관리자 권한이 필요함
- 일반 사용자 권한으로 실행되어 권한 부족 에러 발생

---

## ✅ 해결 방법

### 방법 1: 코드 서명 비활성화 (권장 - 개발/테스트용)

개발 단계에서는 코드 서명이 필요하지 않습니다. `package.json`에서 코드 서명을 비활성화하세요.

**이미 적용됨:** `package.json`의 `win` 설정에 `"sign": null`을 추가했습니다.

### 방법 2: 관리자 권한으로 실행

1. **CMD를 관리자 권한으로 실행**
   - Windows 검색에서 "cmd" 입력
   - "명령 프롬프트" 우클릭 → "관리자 권한으로 실행"

2. **빌드 명령어 실행**
   ```bash
   cd D:\Scripts\RefBoard
   npm run electron:build
   ```

### 방법 3: 캐시 삭제 후 재시도

```bash
# electron-builder 캐시 삭제
rmdir /s /q "%LOCALAPPDATA%\electron-builder\Cache\winCodeSign"

# 다시 빌드
npm run electron:build
```

```bash

```

## 📝 참고사항

### 코드 서명이란?

- Windows 앱을 배포할 때 코드 서명을 사용하면 사용자에게 "알 수 없는 게시자" 경고를 피할 수 있습니다
- 코드 서명을 하려면 코드 서명 인증서가 필요합니다 (유료)
- 개발/테스트 단계에서는 코드 서명 없이도 정상 작동합니다

### 프로덕션 배포 시

나중에 실제 배포할 때는:
1. 코드 서명 인증서 구매
2. `package.json`에서 `"sign": null` 제거
3. 인증서 경로 설정
