# 지금 해야 할 일 (단계별)

## ✅ 1단계: 다시 빌드하기

CMD 창에서 다음 명령 실행:

```bash
cd D:\Scripts\RefBoard
npm run build:web
```

**왜 다시 빌드하나요?**
- `vite.config.web.ts`의 `base` 설정을 `/`로 변경했기 때문입니다
- 올바른 경로로 빌드해야 합니다

---

## ✅ 2단계: 파일 다시 복사하기

### Windows 탐색기 사용 (가장 쉬움)

1. **빌드된 파일 폴더 열기**
   - `D:\Scripts\RefBoard\dist-web` 폴더를 탐색기에서 엽니다

2. **모든 파일 선택**
   - `Ctrl + A` (모두 선택)

3. **복사**
   - `Ctrl + C` (복사)

4. **리포지토리 폴더로 이동**
   - `D:\Scripts\refboard-website` 폴더를 탐색기에서 엽니다

5. **기존 파일 덮어쓰기**
   - `Ctrl + V` (붙여넣기)
   - "파일 바꾸기" 확인 창이 나오면 "모두 바꾸기" 클릭

---

CMD 창에서:

- `404.html`, `assets/`, `carbon-ad.html` (새 파일)

---

## ✅ 4단계: 변경사항 저장하기 (커밋)

```bash
# 모든 변경사항 추가
git add .

# 저장 (커밋)
git commit -m "Deploy RefBoard app to root"
```

**무슨 일이 일어나는가?**
- 변경사항이 로컬에 저장됩니다
- 아직 GitHub에는 업로드되지 않았습니다

---

## ✅ 5단계: GitHub에 업로드하기 (푸시)

```bash
git push origin main
```

**무슨 일이 일어나는가?**
- 로컬의 변경사항이 GitHub에 업로드됩니다
- GitHub 사용자 이름과 비밀번호(또는 토큰)를 입력해야 할 수 있습니다

**인증이 필요한 경우:**
- GitHub 사용자 이름 입력
- 비밀번호 대신 Personal Access Token 사용
- 토큰이 없다면: GitHub → Settings → Developer settings → Personal access tokens → Generate new token

---

## ✅ 6단계: GitHub Pages 확인하기

1. **리포지토리 페이지 접속**
   - 브라우저에서 `https://github.com/trackk81/refboard-website` 접속

2. **Settings > Pages 확인**
   - 리포지토리 상단의 "Settings" 클릭
   - 왼쪽 메뉴에서 "Pages" 클릭
   - Source가 "Deploy from a branch"로 설정되어 있는지 확인
   - Branch가 "main"인지 확인
   - Folder가 "/ (root)"인지 확인

3. **배포된 사이트 확인**
   - 몇 분 후 (보통 1-5분)
   - `https://trackk81.github.io/refboard-website/` 접속
   - RefBoard 앱이 로드되는지 확인

## 🆘 문제 해결

### 빌드가 안 되는 경우
- CMD를 사용하세요 (PowerShell 대신)
- `npm install`을 먼저 실행해보세요

### 파일 복사가 안 되는 경우
- Windows 탐색기를 사용하세요
- `dist-web` 폴더에 파일이 있는지 확인하세요

### git push가 안 되는 경우
- GitHub 인증 확인
- Personal Access Token 사용
- 인터넷 연결 확인

### 페이지가 로드되지 않는 경우
- GitHub Pages 설정 확인
- 브라우저 콘솔(F12)에서 에러 확인
- 몇 분 더 기다려보세요 (배포에 시간이 걸릴 수 있습니다)
